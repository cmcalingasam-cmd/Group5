Folder Structure
Purpose of Folder Structure
src/components/: Holds all modular and reusable components to encourage code reuse.
src/services/: Separates business logic from UI to maintain clean architecture.
src/utils/: Stores helper functions to avoid code duplication.
docs/: Keeps project documentation organized.
tests/: Dedicated folder for test cases, ensuring easier maintenance.
Component Layering Strategy
Presentation Layer: Handles UI and interaction (components).
Business Logic Layer: Processes data and implements application logic (services).
Utility Layer: Provides shared helper functions (utils).
This strategy ensures separation of concerns, easier maintenance, and scalability for future development.

Tools Used
VS Code: IDE for editing and managing project files.
Git: Version control to track changes and collaborate effectively.
